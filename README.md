Смысл RAG
RAG - это архитектура, которая сочетает:

Retrieval (поиск) - находит релевантные документы из базы знаний

Augmented (усиление) - обогащает промпт контекстом из найденных документов

Generation (генерация) - генерирует ответ на основе контекста

Основные преимущества:

Снижает галлюцинации модели

Позволяет работать с актуальными данными

Обеспечивает проверяемость ответов

Не требует тонкой настройки модели
